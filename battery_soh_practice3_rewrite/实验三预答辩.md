# 实验三预答辩：锂离子电池 SOH 预测

## 1. 实验任务与数据来源

本实验的目标是使用深度学习方法预测锂离子电池健康状态 SOH。SOH 定义为：

```text
SOH(%) = 当前放电容量 / 额定容量 2.0Ah * 100
```

本实验使用 NASA PCoE Battery Aging 数据集中的 4 块电池：`B0005`、`B0006`、`B0007`、`B0018`。原始数据为 `.mat` 格式，每个文件中包含 `charge`、`discharge`、`impedance` 三类循环。本实验只使用 `discharge cycle`，因为放电阶段包含容量 `Capacity`，可以直接用于计算 SOH 标签。

## 2. 数据读取与预处理

原始 `.mat` 文件结构较复杂，因此我先封装了统一的数据读取逻辑。程序会从 `data/BatteryAgingARC.zip` 自动解压到 `data/extracted/`，然后读取四个 `.mat` 文件。

每个 discharge cycle 中提取以下字段：

```text
Time
Voltage_measured
Current_measured
Temperature_measured
Capacity
ambient_temperature
```

数据清洗主要包括以下步骤：

1. 删除解析失败的 cycle。
2. 删除时间、电压、电流、温度长度不一致的数据。
3. 删除非有限值，例如 NaN 和 inf。
4. 删除容量明显异常的数据，例如容量小于 0.5Ah 或大于 2.2Ah。
5. 检查时间轴，要求时间递增，并去除重复时间点。
6. 增加局部容量尖峰检测，剔除与邻近 cycle 相比明显异常回升的容量点。

局部容量尖峰检测的阈值不是手工指定某个 cycle，而是使用局部窗口半径 4 的邻域中位数和 MAD。判定规则为：

```text
当前容量 > 邻域中位数 + max(0.04Ah, 3 * 1.4826 * MAD)
```

其中 `MAD` 是邻域容量的 median absolute deviation，`1.4826 * MAD` 用于近似稳健标准差。设置 `0.04Ah` 的下限，是为了避免在局部波动极小时过度删除正常小波动。

最终清洗结果如下：

| 电池 | 原始 discharge 数 | 保留 discharge 数 | 局部容量尖峰剔除数 |
| --- | ---: | ---: | ---: |
| B0005 | 168 | 166 | 2 |
| B0006 | 168 | 165 | 3 |
| B0007 | 168 | 167 | 1 |
| B0018 | 132 | 131 | 1 |

这里额外处理容量尖峰的原因是：电池容量整体应随循环次数下降，虽然真实实验中会有小幅波动，但如果某个点相对邻近 cycle 出现明显孤立回升，会影响 SOH 标签质量，进而影响模型训练。

被剔除点的具体位置如下：

| 电池 | cycle_index | global_cycle_index | 容量 Ah | 局部中位数 Ah | 偏离量 Ah | 阈值 Ah |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| B0005 | 30 | 85 | 1.8518 | 1.8116 | 0.0402 | 0.0400 |
| B0005 | 89 | 312 | 1.6058 | 1.5282 | 0.0776 | 0.0400 |
| B0006 | 30 | 85 | 1.9248 | 1.8557 | 0.0691 | 0.0514 |
| B0006 | 89 | 312 | 1.5936 | 1.4675 | 0.1261 | 0.1034 |
| B0006 | 120 | 432 | 1.4051 | 1.3607 | 0.0444 | 0.0400 |
| B0007 | 89 | 312 | 1.6888 | 1.5961 | 0.0928 | 0.0400 |
| B0018 | 39 | 101 | 1.6761 | 1.6249 | 0.0512 | 0.0431 |

## 3. 特征构造

每个 discharge cycle 被转换成一行结构化特征。特征主要来自电压、电流、温度和时间曲线，包括：

```text
duration_s
voltage_start / voltage_end / voltage_mean / voltage_std
current_mean / current_abs_mean / current_std
temp_start / temp_end / temp_mean / temp_rise / temp_range
time_to_4.1V / 4.0V / 3.9V / 3.8V / 3.7V / 3.6V / 3.5V
voltage_at_25pct_time / 50pct_time / 75pct_time
temp_at_25pct_time / 50pct_time / 75pct_time
early_voltage_mean
early_voltage_slope
cycle_index
global_cycle_index
ambient_temperature
```

其中 `capacity_ah` 不作为模型输入，因为它本身就是 SOH 标签的来源。同时，`charge_ah_integral` 和 `energy_wh` 也不进入模型训练，因为这些积分特征与容量关系过强，可能造成标签泄露。它们可以保留在数据表中用于分析，但不能作为模型输入特征。

## 4. 三种数据划分方式

本实验按照任务书要求实现了三种划分方式。

### A. 单电池随机划分

在单个电池数据上随机划分：

```text
60% 训练集
20% 验证集
20% 测试集
```

这种方式主要检验模型在同一电池数据分布内的插值能力。

### B. 单电池按 cycle 顺序划分

在单个电池中按 cycle 顺序划分：

```text
前 60% cycle 用于训练和验证
后 40% cycle 用于测试
```

这种场景更接近真实预测任务，因为实际应用中通常是用早期数据预测后期衰退趋势。

这里验证集不是从后 40% 测试段中抽取的，而是在前 60% development 段内部按时间顺序再划出最后 20% 作为验证集。因此 B 类整体约为：前 48% 训练、接下来的 12% 验证、最后 40% 测试。

### C. 跨电池迁移学习

跨电池迁移学习的设置为：

```text
源电池数据 + 目标电池前 10% 数据用于训练
目标电池后 90% 数据用于测试
```

严格来说，这不是零样本迁移学习，而是“源电池全量 + 目标电池少量早期数据”的少样本校准或迁移微调设置。这样设计的原因是，不同电池之间衰退速度不同，完全不使用目标电池数据时泛化难度很高；加入目标前 10% cycle 可以模拟实际场景中已有少量早期运行数据的情况。

这里不是只选择少量迁移组合，而是对四个电池做全部有向组合，共 12 组：

```text
B0005 -> B0006, B0007, B0018
B0006 -> B0005, B0007, B0018
B0007 -> B0005, B0006, B0018
B0018 -> B0005, B0006, B0007
```

这样可以更全面地观察不同源电池对不同目标电池的迁移效果。

## 5. 特征选择方法

特征选择使用 Pearson Correlation Coefficient，即 PCC。为了避免测试集信息泄露，我只在训练集上计算每个特征与 SOH 的相关系数。

排序方式是按相关系数绝对值从大到小排列，然后选择 Top K 特征作为模型输入。默认选择前 8 个特征。

通常排名靠前的特征包括：

```text
time_to_3p6v_s
time_to_3p5v_s
time_to_3p7v_s
early_voltage_slope
voltage_mean
cycle_index
global_cycle_index
duration_s
```

这些特征合理的原因是：随着电池老化，容量下降，同样放电条件下到达某个电压阈值的时间会缩短；电压平台和早期电压斜率也会发生变化；`cycle_index` 反映退化过程，但跨电池泛化时需要谨慎，因为不同电池的衰退速度不完全一致。因此在特征选择阶段，`cycle_index` 的 PCC 排名虽高，但它在域迁移场景下的稳定性仍需要通过消融实验验证。

## 6. 模型设计

模型使用 MLP 回归网络，结构如下：

```text
输入层：Top K 个特征
隐藏层 1：Linear(K -> 64) + ReLU + Dropout(0.05)
隐藏层 2：Linear(64 -> 32) + ReLU
输出层：Linear(32 -> 1)
```

选择 MLP 的原因是：本实验输入不是原始长时间序列，而是每个 cycle 提取出的结构化统计特征。样本量也较小，如果直接使用复杂的 LSTM 或 Transformer，容易过拟合。MLP 参数量较小，训练稳定，也便于解释 PCC 特征选择和模型输入之间的关系。

训练策略如下：

```text
Optimizer: AdamW
Loss: SmoothL1Loss
Early stopping: 根据验证集 loss 早停
输入和标签都做标准化
```

使用 `SmoothL1Loss` 是因为它相比 MSE 对异常点更稳健；`AdamW` 可以减少权重过拟合；早停可以防止训练过久导致验证集性能下降。

## 7. 实验结果

完整实验一共 20 组：

```text
A 随机划分：4 组
B 时序划分：4 组
C 跨电池迁移：12 组
```

三类平均结果如下：

| 类别 | MAE | RMSE | R2 |
| --- | ---: | ---: | ---: |
| A | 0.2217 | 0.2949 | 0.9987 |
| B | 0.7871 | 0.8880 | 0.8728 |
| C | 3.2211 | 3.4955 | 0.7638 |

从结果可以看出，A 类随机划分效果最好，因为训练集和测试集来自同一电池，并且随机混合了早期和后期 cycle，本质上更接近插值任务。

B 类难度更高，因为模型需要用前期数据预测后期衰退，是一个时序外推问题，所以误差比 A 类更大。

C 类跨电池迁移最难，因为不同电池虽然实验条件类似，但衰退曲线、容量下降速度和截止电压存在差异。某些源电池到目标电池的迁移效果较差，说明源域和目标域分布差异明显。

## 8. 迁移学习结果分析

C 类中表现较好的组合包括：

| 迁移组合 | MAE | R2 |
| --- | ---: | ---: |
| B0018 -> B0005 | 0.7191 | 0.9883 |
| B0018 -> B0007 | 0.9202 | 0.9824 |
| B0007 -> B0005 | 1.0493 | 0.9808 |

表现较差的组合包括：

| 迁移组合 | MAE | R2 |
| --- | ---: | ---: |
| B0018 -> B0006 | 6.7627 | 0.6011 |
| B0006 -> B0005 | 6.4978 | 0.4304 |
| B0006 -> B0007 | 6.0842 | 0.2774 |

这说明迁移学习不是任意源电池都有效。源电池和目标电池的退化模式越接近，迁移效果越好；如果源电池衰退趋势与目标电池差异较大，只使用目标前 10% 数据可能不足以完成很好校准。

## 9. 补充消融实验

### 9.1 容量尖峰剔除消融

为回应“删除容量回升点是否会让测试指标变好”的问题，我补充了剔除容量尖峰与保留容量尖峰的对比实验。结果如下：

| 类别 | 剔除尖峰 MAE | 保留尖峰 MAE | 保留 - 剔除 |
| --- | ---: | ---: | ---: |
| A | 0.2217 | 0.1832 | -0.0385 |
| B | 0.7871 | 0.9922 | 0.2051 |
| C | 3.2211 | 3.1667 | -0.0544 |

这个结果说明，容量尖峰剔除并不是单向提升指标，而是具有场景依赖性。A 类随机划分和 C 类跨电池平均结果在保留尖峰时略低，说明这些点对插值型评估和部分迁移组合不一定造成负面影响；但 B 类时序外推在保留尖峰时 MAE 从 0.7871 上升到 0.9922，说明局部容量回升会干扰模型学习连续衰退趋势，损害对未来 cycle 的预测。因此我在报告中把它作为标签质量控制和敏感性分析，而不是简单声称“剔除后模型一定更好”。答辩时可以强调：我保留了 `outputs/11_ablation_capacity_spike_cleaning.csv`，用实际对比说明清洗策略的影响。

### 9.2 C 类迁移去除 cycle 序号特征消融

`cycle_index` 和 `global_cycle_index` 在单电池 A/B 场景中合理，因为它们描述老化进度；但在 C 类跨电池迁移中可能编码源电池特定的退化节奏。为此，我补充了 C 类去除这两个特征的消融实验。

| C 类迁移设置 | MAE | RMSE | R2 |
| --- | ---: | ---: | ---: |
| 保留 cycle 序号特征 | 3.2211 | 3.4955 | 0.7638 |
| 去除 cycle 序号特征 | 3.9387 | 4.2624 | 0.6849 |

从平均结果看，去除序号特征后 C 类 MAE 上升 0.7177，R2 下降约 0.0789，说明序号特征确实帮助了一部分迁移组合。但分组合看并非所有情况都依赖序号特征：`B0005 -> B0007` 去除后 MAE 从 2.0611 降到 1.3683，`B0007 -> B0005` 去除后 MAE 从 1.0493 降到 0.5399。这说明跨电池迁移中序号特征具有两面性：它能提供老化进度信息，也可能引入源电池退化节奏偏差，在某些迁移对上形成负迁移风险。

因此更严谨的表述是：主实验保留序号特征以满足单电池预测和整体特征选择；同时在 C 类迁移中报告去除序号特征的消融结果，承认跨电池泛化不能完全依赖 cycle 序号。

## 10. 实验中遇到的问题与解决

第一个问题是 `.mat` 数据结构复杂。解决方法是封装统一读取函数，只提取 discharge cycle，并对每个 cycle 解析时间、电压、电流、温度和容量。

第二个问题是容量曲线存在局部异常点。解决方法是在基础清洗后，使用局部中位数和 MAD 方法检测孤立容量尖峰，避免异常标签影响模型。

第三个问题是特征泄露。容量本身和 SOH 直接相关，所以不能把 `capacity_ah` 作为输入；容量积分类特征也可能让模型间接看到标签，所以从模型输入中排除。

第四个问题是随机划分结果过于理想。解决方法是同时设计 B 类时序外推和 C 类跨电池迁移，用更接近真实应用的方式评估模型。

第五个问题是小样本训练不稳定。解决方法是固定随机种子、标准化输入输出、使用 `SmoothL1Loss` 和早停机制。

## 11. 答辩总结

本实验完成了从 NASA 原始 `.mat` 数据读取、清洗、特征构造、PCC 特征选择、MLP 建模，到三种划分方式下训练和测试的完整流程。

最终结果说明：

```text
随机划分下模型预测精度最高；
时序划分更能体现未来 cycle 预测能力；
跨电池迁移最困难，但可以反映模型的泛化能力。
```

整体来看，基于放电曲线统计特征和 MLP 的方法可以较好完成 SOH 预测任务，但跨电池泛化仍然是主要难点。本实验通过设计难度递进的三种评估协议，并补充容量尖峰清洗和 cycle 序号特征两类消融，较为系统地揭示了基于统计特征的 MLP 方法在 SOH 预测中的能力边界与主要瓶颈。后续如果继续改进，可以尝试加入更多电池数据、使用领域自适应方法，或者直接用时间序列模型提取放电曲线动态特征。
