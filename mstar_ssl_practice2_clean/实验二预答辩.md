# 实验二预答辩

## 一、实践目的

本次实践二的主题是深度半监督学习，核心目标是验证两个问题：

1. 当训练标签数量减少时，深度分类网络的性能会受到多大影响。
2. 引入半监督学习方法后，是否能够利用无标签样本恢复一部分分类精度。

本实验使用 MSTAR SAR 目标分类数据集，任务是对 10 类地面目标进行图像分类。实验环境为 PyTorch，分类网络选择 SmallResNet。

选择 SmallResNet 的原因是残差结构训练稳定，能够缓解深层网络训练中的退化问题，同时参数规模适中，适合本实验这种中小规模 SAR 图像分类任务。

## 二、实验设计

实验共设计三组对照：

| 实验名称 | 标签使用方式 | 实验目的 |
| --- | --- | --- |
| 全标签监督 | 使用全部训练标签 | 作为分类性能参考上限 |
| 10% 标签监督 | 每类只保留约 10% 有标签样本 | 验证标签减少对模型性能的影响 |
| FixMatch 半监督 | 使用同样 10% 有标签样本，并使用剩余训练图像作为无标签数据 | 验证半监督方法能否恢复分类精度 |

这三组实验对应实践要求中的两个核心内容：

1. 标签有限对于深度学习网络训练性能的影响。
2. 半监督方法解决有限标签问题。

## 三、数据处理

实验开始前，将 `MSTAR.zip` 解压到项目目录下的标准数据结构：

```text
data/MSTAR/
  mstar-train/
  mstar-test/
```

其中 `mstar-train` 用于训练，`mstar-test` 用于测试。每个类别单独一个文件夹。

读取图像时，所有样本统一转为单通道灰度图，并缩放到 `128×128`。这样可以保证输入尺寸和通道数一致，便于送入网络训练。

监督训练中使用弱增强，包括随机裁剪、水平翻转和归一化。FixMatch 半监督训练中，对无标签样本同时生成弱增强视图和强增强视图，用于伪标签一致性训练。

## 四、有限标签划分

有限标签实验不是直接从全部训练样本中随机抽取 10%，而是采用按类别分层采样。

这样做的原因是：如果直接随机抽样，某些类别可能被抽到的样本过少，甚至没有样本，会导致有限标签实验的类别分布不稳定。分层采样可以保证每一类都保留一定数量的有标签样本，使对照实验更加公平。

最终 10% 标签实验中：

| 数据类型 | 数量 |
| --- | ---: |
| 有标签样本 | 268 |
| 无标签样本 | 2478 |

在 10% 标签监督实验中，2478 张无标签样本不参与训练；在 FixMatch 半监督实验中，这些样本作为无标签数据参与一致性训练。

## 五、模型与训练设置

本实验使用 SmallResNet 作为分类网络。输入为 `1×128×128` 的 SAR 灰度图，输出为 10 类目标的分类 logits。

为了保证实验对照公平，三组实验中的监督信号 batch size 进行了统一设置：

| 实验名称 | 模式 | 有标签 batch size | μ | 无标签 batch size |
| --- | --- | ---: | ---: | ---: |
| 全标签监督 | supervised | 64 | - | - |
| 10% 标签监督 | supervised | 64 | - | - |
| FixMatch 半监督 | fixmatch | 64 | 2 | 128 |

这里需要特别说明：在 FixMatch 代码中，`batch-size` 表示有标签 batch size，而无标签 batch size 由 `batch-size × μ` 得到。

因此本实验将 10% 标签监督和 FixMatch 的有标签 batch size 都设置为 64。FixMatch 额外设置 `μ=2`，所以每个训练 step 同时使用 64 张有标签样本和 128 张无标签样本。

这样做可以保证两组有限标签实验的监督信号规模一致，主要区别只在于 FixMatch 是否额外利用无标签数据，实验结论更容易解释。

## 六、半监督方法原理

本实验采用 FixMatch 半监督学习方法。FixMatch 的核心思想是伪标签和一致性正则。

具体过程如下：

1. 对无标签图像生成弱增强视图。
2. 模型对弱增强视图进行预测。
3. 如果最大类别概率不低于阈值 `0.95`，就将该预测类别作为伪标签。
4. 对同一张无标签图像生成强增强视图。
5. 要求模型在强增强视图上也预测出相同类别。

训练目标可以写为：

```text
L = Lx + λu Lu
```

其中：

| 符号 | 含义 |
| --- | --- |
| `Lx` | 有标签样本的交叉熵损失 |
| `Lu` | 无标签样本的一致性损失 |
| `λu` | 无标签损失权重，本实验设置为 1.0 |

FixMatch 的直观理解是：模型先通过少量有标签样本学习基本分类能力，再利用高置信度预测为无标签样本生成伪标签，从而挖掘无标签数据中的类别结构。

## 七、实验运行方式

实验原始脚本 `scripts/run_all.sh` 中包含三组训练任务。为了节省 GPU 时间，实际运行时没有串行执行，而是并行启动三组实验：

```bash
python train.py --mode supervised --data-root data/MSTAR --label-ratio 1.0 --epochs 80 --batch-size 64 --out runs/01_supervised_full

python train.py --mode supervised --data-root data/MSTAR --label-ratio 0.1 --epochs 80 --batch-size 64 --out runs/02_supervised_10percent

python train.py --mode fixmatch --data-root data/MSTAR --label-ratio 0.1 --epochs 80 --batch-size 64 --mu 2 --threshold 0.95 --lambda-u 1.0 --out runs/03_fixmatch_10percent
```

三组训练完成后，统一生成答辩图表和报告素材：

```bash
python tools/make_presentation_assets.py --runs runs --out presentation_assets
```

每组实验都会保存以下结果文件：

| 文件 | 内容 |
| --- | --- |
| `metrics.json` | 最优测试精度、最优 epoch、样本数量和训练耗时 |
| `history.csv` | 每个 epoch 的训练损失、测试损失和测试精度 |
| `confusion_matrix.csv` | 测试集混淆矩阵 |
| `best.pt` | 最优模型权重 |

## 八、实验结果

三组实验最终结果如下：

| 实验名称 | 最优测试精度 | 最优 epoch | 末轮测试精度 | 训练耗时 |
| --- | ---: | ---: | ---: | ---: |
| 全标签监督 | 97.07% | 71 | 96.49% | 310.01s |
| 10% 标签监督 | 73.65% | 54 | 58.23% | 121.57s |
| FixMatch 半监督 | 82.60% | 68 | 75.96% | 376.40s |

从结果可以看出，全标签监督精度最高，达到 97.07%，说明 SmallResNet 在完整标签条件下可以较好地完成 MSTAR 分类任务。

当训练标签减少到约 10% 后，最优测试精度下降到 73.65%，相比全标签监督下降约 23.42 个百分点。这说明深度网络对标注样本数量比较敏感，有限标签条件下容易出现泛化能力下降。

加入 FixMatch 后，最优测试精度提升到 82.60%，相比 10% 标签监督提升约 8.95 个百分点。这说明在有标签样本数量不变的情况下，额外引入无标签样本可以帮助模型恢复一部分性能。

不过 FixMatch 仍然没有达到全标签监督的 97.07%，说明半监督学习能够缓解标签不足问题，但不能完全替代真实标签。

## 九、训练曲线分析

从训练集和测试集损失、精度曲线可以观察到：

1. 全标签监督的训练过程较稳定，测试精度上升较快，最终保持在较高水平。
2. 10% 标签监督由于标注样本较少，训练集上容易拟合，但测试集表现不稳定，末轮测试精度明显低于最优测试精度，体现出过拟合风险。
3. FixMatch 通过无标签一致性约束改善了泛化能力，整体测试精度高于单纯 10% 标签监督。

这些曲线说明，仅依靠少量标签训练时，模型容易过度依赖有限标注样本；引入无标签样本后，可以在一定程度上稳定训练并提升测试性能。

## 十、混淆矩阵分析

混淆矩阵中，全标签监督模型的对角线最明显，说明大多数类别都能被正确识别。

10% 标签监督实验的混淆明显增加，部分类别之间误判更多。这说明标签数量减少后，模型难以充分学习不同目标类别之间的判别特征。

FixMatch 的混淆矩阵相比 10% 标签监督有所改善，对角线更加集中，说明无标签数据提供了额外的分布信息，有助于减少部分类别混淆。

## 十一、答辩时可强调的处理细节

### 1. 为什么采用分层采样？

因为 MSTAR 是多类别分类任务，如果直接随机抽取 10% 训练样本，可能造成部分类别样本过少，影响实验公平性。分层采样可以保证每个类别都保留一定比例的有标签样本。

### 2. 为什么要统一有标签 batch size？

本实验要比较的是“同样有限标签条件下，是否使用无标签数据”的差异。如果 10% 标签监督和 FixMatch 的有标签 batch size 不一致，那么性能差异可能来自监督信号规模不同。

因此将两者的有标签 batch size 都设置为 64。FixMatch 额外使用 128 张无标签样本，体现半监督方法利用额外数据的特点。

### 3. 为什么 FixMatch 没有达到全标签监督？

FixMatch 依赖模型产生的伪标签，而伪标签可能存在错误。虽然阈值 `0.95` 可以过滤低置信度预测，但也会降低无标签样本利用率。

另外，SAR 图像与普通自然图像不同，目标散射结构对增强方式比较敏感。如果强增强过强，可能破坏目标特征。因此 FixMatch 可以恢复一部分精度，但很难完全等同于真实标签监督。

### 4. 为什么选择 FixMatch？

FixMatch 实现相对清晰，同时包含半监督学习中常见的两个关键思想：伪标签和一致性正则。它适合用于说明半监督学习如何利用无标签样本缓解标签不足问题。

## 十二、总结

本实验完成了实践二要求的两个核心内容：

1. 通过全标签监督和 10% 标签监督对比，验证了标签数量减少会明显降低深度分类网络性能。
2. 通过 FixMatch 半监督实验，验证了无标签数据能够在有限标签条件下提升模型分类精度。

实验结果表明，SmallResNet 在全标签条件下可以达到 97.07% 的测试精度；当标签减少到约 10% 后，精度下降到 73.65%；加入 FixMatch 后，精度恢复到 82.60%。

因此可以得出结论：半监督学习能够有效缓解标签不足带来的性能下降，但其效果仍受伪标签质量、增强策略和阈值设置影响。
