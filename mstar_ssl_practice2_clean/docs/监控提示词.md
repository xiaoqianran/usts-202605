# MSTAR SSL 网格实验 监控 / 看管 Prompt（给 Codex / 其他 Grok 实例）

**目标**：在 GPU 空闲时，确保监督学习 16 组网格（fixed + scaled）全部跑完，并与已完成的 FixMatch 8+8 形成完整对标。任何中断或 OOM 都要自动恢复/降并发重跑。

## 1. 关键目录与脚本（cd 到这里再操作）

```bash
cd /teamspace/studios/this_studio/mstar_ssl_practice2_clean
```

核心脚本（已全部中文化 + skip 逻辑 + 实时监控）：

- `scripts/monitor_training.sh --once` 或 `--watch 30`  
  **中文表格输出**：GPU 状态 + 每个 grid 的实验目录 | 状态（已完成/运行中）| 轮次 | 测试精度 | 日志状态（正常/OOM崩溃/错误）

- `scripts/run_supervised_grid.sh`  
  智能跳过：只要某 `supervised_*_bXX/metrics.json` 存在就跳过。支持 `LR_POLICY=fixed|scaled`，启动和结束提示均为中文。

- `scripts/babysit_supervised_grids.sh data/MSTAR`  
  一键顺序跑完固定学习率 → 线性缩放学习率 两套（推荐，后台挂起最方便）

- 旧的 `scripts/run_fixmatch_grid.sh` 也支持同样 env 变量。

## 2. 标准操作流程（按优先级）

1. **先看全局状态**（每次接管必做）：
   ```bash
   bash scripts/monitor_training.sh --once
   # 会看到漂亮的中文表格：
   # === GPU 状态 @ 22:15:xx ===
   #   显存: 14380 MB / 81559 MB | 利用率=100% | 功耗=413 W
   # === runs_supervised_grid_fixed ===
   #   实验目录                     | 状态   | 轮次 | 测试精度 | 日志状态
   #   supervised_full_b32          | 运行中 |   26 |   57.11% | 正常
   #   ...
   nvidia-smi --query-gpu=memory.used,memory.total,utilization.gpu,power.draw --format=csv
   ```

2. **如果 supervised 两套都已完成**（各 8 组、都有 metrics.json、summary 存在）：
   - 直接 `cat runs_supervised_grid_fixed/summary/*.md runs_supervised_grid_scaled/summary/*.md`
   - 确认 16 行数据齐全后，报告“全部完成”。

3. **如果 fixed 未完成**：
   ```bash
   LR_POLICY=fixed PARALLEL_JOBS=3 EPOCHS=80 bash scripts/run_supervised_grid.sh data/MSTAR
   ```
   （PARALLEL_JOBS=3 更稳，H100 80GB 也能扛 128+128 的 FixMatch，但监督 4 个同时很安全）

4. **fixed 完成后立刻接 scaled**（或直接用 babysit）：
   ```bash
   LR_POLICY=scaled PARALLEL_JOBS=3 EPOCHS=80 bash scripts/run_supervised_grid.sh data/MSTAR
   ```

5. **任何时候发现 OOM / crash**（monitor 里出现 "OOM/CRASH" 或 acc 异常低）：
   - `pkill -f "train.py --mode supervised"` 杀掉所有
   - 删掉有问题的 run 目录（`rm -rf runs_supervised_grid_*/supervised_*_b128` 等）
   - 把 PARALLEL_JOBS 降到 2，重跑该策略

6. **后台挂 babysit（最省心）**：
   ```bash
   nohup bash scripts/babysit_supervised_grids.sh data/MSTAR \
     > logs/babysit_sup_$(date +%Y%m%d_%H%M).log 2>&1 &
   tail -f logs/babysit_sup_*.log
   ```

## 3. 判断“完成”的唯一标准

- 目录下存在 `.../metrics.json`（train.py 跑完 80 epoch 后才会写）
- summary/*.md 里正好 8 行数据
- monitor 表格里全部显示「已完成」，且日志状态无「OOM崩溃」

monitor 脚本已经把这些检查自动化了（会显示「进度: 8 / 8 已完成」）。
