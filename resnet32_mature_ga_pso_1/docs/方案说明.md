# 方案说明：重要性引导 GA/PSO ResNet32 结构化通道压缩

## 核心思想

保持 ResNet32 的深度结构不变，只改变 15 个 residual block 的输出通道数。通道配置由 GA 和 PSO 自动搜索。

## 为什么比 stage-level 搜索成熟

旧方案只搜索 3 个变量：

```text
[c1, c2, c3]
```

新方案搜索 15 个变量：

```text
[c1_1,...,c1_5,c2_1,...,c2_5,c3_1,...,c3_5]
```

这样可以表达“同一个 stage 内不同 block 冗余程度不同”的情况。

## 重要性引导

如果提供 baseline checkpoint，代码会读取标准 ResNet32 中每个 block 的第二个 BN 层 `gamma`：

```text
importance_i = mean(abs(gamma_i))
```

重要性高的 block 更倾向保留较大通道数；重要性低的 block 更倾向压缩。

## 维度不匹配处理

当相邻 block 的通道数不同，或者 stride=2 发生空间下采样时，使用：

```text
1x1 Conv + BN projection shortcut
```

这样可以让任意 block-level 通道配置稳定前向传播。

## 实验组建议

| 组别 | 说明 |
|---|---|
| Baseline | 标准 ResNet32 |
| Manual | 手动压缩配置，例如中后层减通道 |
| GA-Block | GA 搜索 block-level 配置 |
| PSO-Block | PSO 搜索 block-level 配置 |
| GA/PSO + KD | 用 baseline teacher 蒸馏恢复精度 |
